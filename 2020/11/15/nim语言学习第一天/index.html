Nim语言
发布日期:   2020-11-15

第一部分 Nim基础

本书的这一部分开始您对Nim编程语言的研究。它并不假定您对Nim有很多了解，因此第$1$章首先介绍了该语言的特性，与其他语言的不同之处以及其在现实世界中的使用方式。第$2$章介绍了任何编程语言中最常用的一些元素（语法，语义和类型系统），并在此过程中教您了用Nim编写简单应用程序的必要基础。

这一章包括：

  • 什么是Nim
  • 为什么你应该学习它

Nim仍然是一种相对较新的编程语言。实际上，您拿着的是其中的第一本书籍。该语言仍未完全完成，但是核心方面（如语法，过程，方法，迭代器，泛型，模板等的语义）都是固定的。尽管它是新颖的，但由于它实现并为用户提供了独特的功能集，因此编程社区对Nim表现出了极大的兴趣。

本章回答在学习Nim之前可能会问的问题，例如为什么要使用它。在本章中，我概述了Nim的一些常见实际用法，将其与其他编程语言进行比较，并讨论了它的一些优缺点。

什么是Nim

Nim是一种通用的编程语言，旨在高效，富有表现力和优雅。这三个目标很难同时实现，因此Nim的设计师为每个目标赋予了不同的优先级，其中效率是最重要的，而优雅则是最小的。

但是，尽管优雅对Nim的设计而言并不重要，但在设计过程中仍会考虑它。因此，该语言本身仍然保持优雅。只有当需要在效率和优雅之间做出权衡时，效率才被优先考虑。

从表面上看，Nim具有许多Python的特征。特别是，Nim语法的许多方面与Python相似，包括使用缩进来界定范围，以及某些操作员倾向于使用单词代替符号的趋势。 Nim还与Python共享与语法无关的其他方面，例如高度用户友好的异常回溯，如下所示：

使用情境

从一开始，Nim就被设计为通用编程语言。因此，它具有广泛的功能，可用于几乎所有软件项目。这使其成为在从Web应用程序到内核的各种应用程序域中编写软件的理想人选。在本节中，我将讨论Nim的功能和编程支持如何在几种用例中应用。尽管Nim几乎可以支持任何应用程序域，但这并不是所有应用程序的正确选择。该语言的某些方面使其比其他类别更适合某些类别的应用程序。这并不意味着某些应用程序无法使用Nim编写；这只是意味着Nim可能不支持最适合编写某些类型的应用程序的代码样式。 Nim是一种编译语言，但是其编译方式很特殊。 Nim编译器编译源代码时，首先将代码转换为C代码。 C是一种古老而又受良好支持的系统编程语言，它可以更轻松，更直接地访问机器的物理硬件。这使Nim非常适合系统编程，允许编写诸如操作系统（OS），编译器，设备驱动程序和嵌入式系统软件之类的项目。物联网（IoT）设备是具有连接到Internet的嵌入式电子设备的物理设备，是Nim的良好目标，这主要归功于Nim的易用性及其系统编程功能。一个利用Nim的系统编程功能进行项目的很好的例子是一个非常简单的操作系统，称为NimKernel，可在GitHub上找到：https://github.com/dom96/nimkernel。

用Nim编写的应用程序非常快。在许多情况下，速度与用C编写的应用程序一样快，比用Python编写的应用程序快13倍以上。效率是最高优先级，某些功能使优化代码变得容易。这与软实时垃圾收集器并驾齐驱，它使您可以指定收集内存所花费的时间。此功能在游戏开发过程中变得很重要，在这种情况下，如果普通的垃圾收集器使用太多的时间来收集内存，则可能会减慢屏幕上帧的渲染速度。它在需要在非常严格的时间范围内运行的实时系统中也很有用。

Nim可以与其他慢得多的语言一起使用，以加速某些对性能至关重要的组件。例如，用Ruby编写的需要某些CPU密集型计算的应用程序可以部分用Nim编写，以获得相当大的速度优势。在科学计算和高速交易等领域，这种提速非常重要。

Nim也很好地支持执行I/O操作的应用程序，例如通过网络读取文件或发送数据。例如，可以使用许多Web框架（例如Jester(https://github.com/dom96/jester))轻松编写Web应用程序。`Nim`的脚本式语法以及强大的异步I/O支持，使轻松快速开发这些应用程序变得容易。

命令行应用程序可以从Nim的效率中大大受益。另外，由于Nim应用程序是经过编译的，因此它们是独立的，因此不需要任何庞大的运行时依赖项。这使得它们的分发非常容易。用Nim编写的此类应用程序之一就是Nimble；它是Nim的软件包管理器，允许用户安装Nim库和应用程序。

这些只是Nim非常适合的几个用例；这当然不是详尽的清单。要记住的另一件事是，在撰写本文时，Nim仍在开发中，尚未达到1.0版。某些功能尚未实现，因此Nim不适用于某些应用程序。例如，Nim包含一个后端，使您可以在Nim中为网页编写JavaScript应用程序。该后端有效，但还不如其他语言成熟。随着时间的推移，这种情况会有所改善。当然，Nim的JavaScript编译能力使其非常适合需要在服务器和浏览器中运行组件的全栈应用程序。这是一个巨大的优势，因为可以轻松地将代码重用于应用程序的浏览器和服务器组件。现在，您对Nim的含义，它的历史以及它特别适合的一些应用程序有了一些了解，让我们看一下Nim的某些功能并讨论其工作原理。

核心特点

在许多方面，Nim都是非常创新的。 Nim的许多功能都无法通过其他任何编程语言找到。如果您喜欢学习新的编程语言，尤其是那些有趣且独特的功能，那么Nim绝对是适合您的语言。在本节中，我们将研究Nim的一些核心功能，尤其是使Nim在其他编程语言中脱颖而出的功能：

  • 一种称为元编程的工具，用于在多种情况下根据您的需要来塑造语言。
  • 样式不敏感的变量，函数和类型名称。通过使用这个有争议的功能，无论标识符是使用camelCase还是snake_case定义的，您都可以按照所需的样式来对待标识符。
  • 具有丰富功能（例如泛型）的类型系统，可使代码易于编写和维护。
  • 编译为C，可使Nim程序高效且可移植。编译本身也非常快。
  • 可以自由选择或完全删除许多不同类型的垃圾收集器。
元程序设计的

Nim最实用且从某种意义上说独特的功能是其广泛的元编程支持。元编程使您可以阅读，生成，分析和转换源代码。这绝不是Nim的发明，但是没有其他编程语言具有如此广泛的元编程能力，同时也很容易像Nim一样容易使用。如果您熟悉Lisp，那么您可能已经有一些元编程经验。通过元编程，您可以将代码视为抽象语法树形式的数据。这使您可以在编译应用程序时操纵现有代码以及生成全新的代码。 Nim中的元编程非常特殊，因为具有良好的元编程功能的语言通常属于Lisp语言族。如果您已经熟悉Java或Python之类的内容，就会发现开始使用Nim比使用Lisp更容易。与Lisp相比，您还将发现学习如何使用Nim的元编程功能更加自然。本书的所有功能，您将在本书第9章中更加详细地了解。元编程提供的主要好处之一是能够删除样板代码。元编程还允许创建领域特定语言（DSL）；例如：

html:
    body:
        p: "Hello World"

该DSL指定了一点HTML代码。根据实现方式的不同，DSL可能会转换为类似于以下内容的Nim代码：

echo("")
echo("    ")
echo("        
Hello World"
)
echo(" ")
echo("")

该Nim代码将导致以下输出：


    
        

            Hello World

使用Nim的元编程，您可以定义DSL并将它们与普通的Nim代码自由混合。这样的语言有很多用例。例如，前面的代码可用于为您的Web应用程序创建HTML模板。元编程是Nim设计的核心。Nim的设计师希望鼓励用户使用元编程，以适应他们的编程风格。例如，尽管Nim确实提供了一些面向对象的编程（OOP）功能，但它没有类定义构造。相反，任何希望以类似于其他语言的方式在Nim中使用OOP的人都应该使用元编程来创建这种构造。

样式不敏感

Nim另一个有趣且可能独特的功能是样式不敏感。程序员要做的最困难的事情之一就是为各种标识符（例如变量，函数和模块）命名。在许多编程语言中，这些名称不能包含空格，因此程序员不得不采用其他方式将多个单词分隔为一个名称。设计了多种不同的方法，其中最流行的是snake_case和camelCase。使用Nim，即使已使用camelCase定义了标识符，也可以使用snake_case，反之亦然。因此，即使您所使用的库的标识符采用不同的样式，您也可以使用自己喜欢的样式编写代码。

snake_case：蛇形命名法

camelCase：驼峰命名法

之所以可行，是因为Nim认为标识符to_upper和toUpper相等。

Nim在比较标识符时，会考虑第一个字符的大小写，但不会忽略标识符的其余字符的大小写，也忽略下划线。结果，由于首字符的大小写不同，所以标识符toUpper和ToUpper不相等。这允许将类型名称与变量名称区分开，因为按照惯例，类型名称应以大写字母开头，变量名称应以小写字母开头。下面的清单显示了一种使用该约定的方案。

强大的类型系统

它们的类型系统是使编程语言彼此区别的众多特征之一。类型系统的主要目的是减少程序中错误的机会。好的类型系统提供的其他好处是某些编译器优化和更好的代码文档。用于对类型系统进行分类的主要类别为静态和动态。大多数编程语言都介于这两种极端之间，并且融合了两种观点。这是因为静态和动态类型系统都需要某些折衷。静态类型在编译时会发现更多错误，但也会降低编写程序的速度。动态类型则相反。

Nim是静态类型的，但与某些静态类型的编程语言不同，它还具有许多使开发速度更快的功能。类型推断就是一个很好的例子：类型可以由编译器解析，而无需您自己写出类型（尽管您可以选择这样做）。因此，您的程序可以没有错误，但不会妨碍您的开发速度。 Nim还合并了一些动态类型检查功能，例如运行时类型信息，该功能允许动态分配功能。

类型系统确保您的程序没有错误的一种方法是验证内存安全性。某些编程语言（例如C）不是安全的内存，因为它们允许程序访问未分配给其使用的内存。其他编程语言是内存安全的，但代价是不允许程序访问内存的低级详细信息，在某些情况下这是必要的。 Nim兼有两者：只要您在程序中不使用任何不安全的类型（例如ptr），它都是内存安全的，但是与C库接口时ptr类型是必需的。支持这些不安全的功能使Nim成为功能强大的系统编程语言。

默认情况下，Nim保护您免受每种类型的内存错误的影响：

  • 数组在编译时进行边界检查，或者在无法进行编译时检查时在运行时进行边界检查，以防止缓冲区溢出和缓冲区过度读取。
  • 对于引用类型，无法使用指针算术，因为它们完全由Nim的垃圾收集器管理；这样可以避免诸如悬空指针之类的问题以及与手动管理内存有关的其他内存问题。
  • 变量始终由Nim初始化为其默认值，以防止变量包含意外和损坏的数据。

最后，Nim类型系统最重要的功能之一就是能够使用通用编程。 Nim中的泛型可以在不牺牲类型安全性的情况下实现大量代码重用。除其他功能外，它们还允许您指定一个函数可以接受多种不同的类型。例如，您可能有一个showNumber过程，该过程在屏幕上同时显示整数和浮点数：

proc showNumber(num: int | float) = echo(num)
showNumber(3.14)
showNumber(42)

在这里，showNumber过程接受int类型或float类型。 |运算符指定可以将int和float都传递给过程。

编译

我在上一节中提到过，Nim编译器首先将源代码编译为C，然后将源代码提供给C编译器。您将在1.1.3节中了解到更多有关此工作原理的信息，但现在，我将讨论该编译模型的许多实际优势。

C编程语言已经非常成熟地确立为系统编程语言，并且已经使用了40多年。 C是最可移植的编程语言之一，具有Windows，Linux，Mac OS，x86，AMD64，ARM和许多其他更晦涩的OS和平台的多种实现。 C编译器支持从超级计算机到微控制器的所有内容。它们也非常成熟，并实现了许多强大的优化功能，这使得C非常高效。Nim充分利用了C的这些方面，包括其可移植性，广泛使用和效率。编译为C还可以轻松使用现有的C和C ++库-您所需要做的就是编写一些简单的包装器代码。您可以使用称为c2nim的工具更快地编写此代码。此工具将C和C++头文件转换为Nim代码，后者包装这些文件。这是非常有益的，因为许多流行的库都是用C和C++编写的。 Nim还使您能够构建与C和C ++兼容的库。如果您希望从其他编程语言中使用库，这将很方便。您将在第8章中学习有关包装C和C ++库的所有信息。Nim源代码也可以编译为Objective C和JavaScript。 Objective C语言主要用于iOS软件开发；通过编译，您可以在Nim中原生编写iOS应用程序。您还可以通过C++编译后端使用Nim开发Android应用程序。 JavaScript是数十亿个网站使用的客户端语言；它有时被称为“网络的汇编语言”，因为它是所有主要网络浏览器都支持的唯一编程语言。通过编译为JavaScript，您可以在Nim中编写用于Web浏览器的客户端应用程序。

您现在可能想知道Nim编译软件的速度有多快。也许您认为这很慢；毕竟，Nim需要首先将源代码翻译成中间语言。但实际上它的速度相当快。例如，由大约100,000行Nim代码组成的Nim编译器在带有2.7 GHz Intel Core i5 CPU的MacBook Pro上编译大约需要12秒。每个编译都会被缓存，因此时间会在初始编译后减少到5秒。

内存管理

C和C ++都要求您手动管理内存，并仔细确保一旦不再需要分配的内容就将其释放。另一方面，Nim使用垃圾收集器为您管理内存。但是在某些情况下，您可能想避免垃圾收集器；许多人认为它们不足以用于某些应用程序域，例如嵌入式系统和游戏。因此，Nim支持许多具有不同应用程序的不同垃圾收集器。垃圾收集器也可以完全删除，从而使您能够自己管理内存。

垃圾回收器

在垃圾收集器之间切换很容易。您只需要在编译期间指定--gc:<gc_name>，并将替换为markandsweep，boehm或none。

这只是Nim最突出特征的一小部分。还有更多的功能：不仅是独特和创新的功能，而且还有现有编程语言中功能的独特组合，这使得Nim整体上确实非常独特。

Nim如何工作

Nim的独特之处之一是其实现。每种编程语言都有一个应用程序形式的实现，该应用程序要么解释源代码，要么将源代码编译为可执行文件。这些实现分别称为解释器和编译器。某些语言可能具有多种实现，但是Nim唯一的实现是编译器。编译器首先将代码转换为另一种编程语言C，然后将其传递给C编译器，然后再将其编译为二进制可执行文件，从而编译Nim源代码。可执行文件包含指示计算机应执行的特定任务的指令，包括原始Nim源代码中指定的任务。图1.2显示了如何将一段Nim代码编译为可执行文件。大多数编程语言的编译器无需执行此额外步骤；他们自己将源代码编译成二进制可执行文件。还有一些根本不编译代码。图1.3显示了不同的编程语言如何将源代码转换为可以执行的代码。

Nim连接到C编译过程，以便编译由它生成的C源代码。这意味着Nim编译器依赖于外部C编译器，例如GCC或Clang。编译的结果是一个可执行文件，该可执行文件特定于其所编译的CPU体系结构和操作系统。这应该使您对如何将Nim源代码转换为可正常工作的应用程序以及该过程与其他编程语言所使用的过程有何不同有一个很好的了解。每次更改Nim源代码时，都需要重新编译。现在，让我们看一下Nim的优点和缺点。

Nim的优点和缺点

了解您为什么要使用某种语言很重要，但了解为什么该语言可能不适用于您的特定用例也同样重要。在本节中，我将把Nim与许多其他编程语言进行比较，重点是比较中通常使用的各种特征和因素。之后，我将讨论Nim仍需要赶上其他语言的一些领域。

优点

在阅读本书时，您可能会想知道Nim与您熟悉的编程语言相比如何。进行比较的方法很多，可以考虑多种因素，包括语言的执行速度，表达能力，开发速度，可读性，生态系统等。本节着眼于其中一些因素，以使您更好地了解Nim的好处。

Nim是高效的

比较中经常使用以编程语言编写的应用程序执行的速度。 Nim的目标之一是效率，因此这是一种非常高效的编程语言也就不足为奇了。C是最有效的编程语言之一，因此您可能想知道Nim如何比较。在上一节中，您了解了Nim编译器首先将Nim代码翻译成中间语言。默认情况下，中间语言是C，这表明Nim的表现与C相似，这是事实。由于此功能，您可以将Nim用作C的完整替代品，还有一些好处：

  • Nim具有与C相似的性能。
  • Nim产生的软件比用C编写的软件更可靠。
  • Nim具有改进的类型系统。
  • Nim支持泛型。
  • Nim实现了元编程的高级形式。

与C相比，Nim中的元编程是唯一的，因为它不使用预处理程序，而是主要编译过程的一部分。通常，您可以期望在Nim中找到许多在C中找不到的现代功能，因此选择Nim作为C的替代品非常有意义。表1.1显示了一个小型基准测试的结果。Nim与C的速度相当，并且比Python快得多。

在此基准测试中，Nim应用程序的运行时与C应用程序的速度相匹配，并且比Python中实现的应用程序快得多。像这样的微型基准测试通常是不可靠的，但是没有太多选择。 Nim的性能与C相当，后者已经是目前最高效的编程语言之一。

Nim可读性强

Nim是一种非常有表现力的语言，这意味着编写对编译器和人类读者都清晰可见的Nim代码很容易。 Nim代码不会被诸如JavaScript和C ++之类的C编程语言的花括号和分号所困扰，也不需要Ruby之类的语言中的do和end关键字。将此表现力的Nim代码与表现力较弱的C ++代码进行比较：

在Nim中实现$0\sim9$的迭代：

for i in 0 ..< 10:
    echo(i)

在C++里面：

#include 
using namespace std;
int main()
{
    for (int i = 0; i < 10; i++)
    {
        cout << i << endl;
    }
    return 0;
}

Nim代码更具可读性，并且更加紧凑。 C++代码包含许多主要函数声明，这些函数在Nim中是完全隐式的。 Nim易于编写，但更重要的是，它也易于阅读。良好的代码可读性很长一段路要走。例如，它使调试更加容易，从而使您可以花更多的时间编写漂亮的Nim代码，从而减少了开发时间。

Nim是独立的

前面已经提到了这一点，但是值得重新描述一下其他语言的比较方式，尤其是为什么某些语言需要运行时的原因。Nim，C，Go，D和Rust等已编译的编程语言会生成一个可执行文件，该可执行文件对于运行编译器的操作系统而言是本机。在Windows上编译Nim应用程序会导致只能在Windows上执行的可执行文件。同样，在Mac OS上进行编译会生成只能在Mac OS上执行的可执行文件。 CPU架构也可以发挥作用：在ARM上进行编译会生成仅与ARM CPU兼容的可执行文件。这是默认情况下的工作方式，但是可以通过称为交叉编译的过程来指示Nim为不同的OS和CPU组合编译可执行文件。当具有所需体系结构或操作系统的计算机不可用或编译时间太长时，通常使用交叉编译。一种常见的用例是针对Raspberry Pi等ARM设备进行编译，而这些设备通常CPU速度很慢。有关交叉编译的更多信息，请参见《 Nim Compiler用户指南》：http://nim-lang.org/docs/nimc.html#cross-compilation。

其中，创建JVM是为了消除对交叉编译的需求。您可能已经听说过“写一次，就可以在任何地方运行”这句话。一个Java应用程序只需要编译一次，编译的结果就是一个保存所有已编译Java类的JAR文件。然后，JVM可以执行JAR文件，以在任何平台和体系结构上执行已编程的操作。这使JAR文件成为与平台和体系结构无关的可执行文件。缺点是，要运行这些JAR文件，必须在用户系统上安装JVM。 JVM是一个很大的依赖项，可能包含错误和安全性问题。但另一方面，它确实允许Java应用程序仅被编译一次。 Python，Ruby和Perl相似。他们还使用虚拟机（VM）执行代码。在Python的情况下，VM用于优化Python代码的执行，但大多数情况下它作为Python解释器的实现细节被隐藏起来。 Python解释器解析代码，确定代码描述的动作，然后立即执行这些动作。没有像Java，C或Nim那样的编译步骤。但是，优点和缺点与JVM的优点和缺点大部分相同；不需要交叉编译，但是要执行Python应用程序，系统需要安装Python解释器。

不幸的是，在许多情况下，虚拟机和解释器导致的问题多于解决的问题。常见的CPU架构和最流行的OS的数量并不多，因此为它们中的每一个进行编译都没有那么困难。相比之下，以解释语言编写的应用程序的源代码通常会分发给用户，并且期望他们安装正确版本的解释器或虚拟机。这会导致很多问题。与分发此类应用程序相关的困难的一个例子是最近发布的Python3。由于它与先前的版本不向后兼容，因此它对最初用Python 2编写的软件造成了许多问题。Python3于2008年发布，在撰写本文时，仍然有为Python 2编写的库无法与Python 3解释器一起使用。2对于编译语言来说这不是问题，因为二进制文件仍将继续工作。Nim的轻量级特性应该使其特别有吸引力，尤其是与本节中提到的某些语言形成对比。

Nim是灵活的

可以编写许多不同的样式的软件。编程范例是编写软件的基本样式，每种编程语言都支持一组不同的范例。您可能已经熟悉其中的一个或多个，并且至少知道什么是面向对象编程（OOP），因为它是许多计算机科学课程中的一部分。 Nim是一种多范式编程语言。与某些流行的编程语言不同，Nim不专注于OOP范例。它主要是一种过程编程语言，并且对OOP，函数式，声明式，并发和其他编程样式提供不同的支持。这并不是说OOP没有得到很好的支持。 OOP作为一种编程风格完全不会被您强迫。 Nim支持常见的OOP功能，包括继承，多态和动态调度。为了让您更好地了解Nim的主要范式，让我们看一下OOP范式和程序范式之间的一个大差异。在OOP范式中，方法和属性绑定到对象，并且方法在其自己的数据结构上运行。在过程范式中，过程是在数据结构上运行的独立实体。这可能使您难以想象，所以让我们看一些代码示例来说明它。

子程序术语

在本小节中，我将介绍方法和过程。这些只是子例程或函数的不同名称。方法是在OOP上下文中使用的术语，过程在程序编程中使用，函数在功能编程中使用

以下代码清单显示了相同的应用程序。第一种是使用OOP风格用Python编写的。第二种是使用程序样式以Nim编写的。

在Python代码中，bark方法位于类定义下。在Nim代码中，bark方法（在Nim中称为过程）并不以与Python代码中相同的方式绑定到Dog类型。它与Dog类型的定义无关。相反，它的第一个参数指定了与其关联的类型。您还可以在Python中实现类似的功能，但不允许您以相同的方式调用bark方法。您将不得不这样调用它：bark(dog)，将dog变量作为第一个参数显式传递给方法。 Nim并非如此，原因是Nim将dog.bark()重写为bark(dog)，使您可以使用传统的OOP样式来调用方法，而不必将它们显式绑定到类。称为统一函数调用语法（UFCS）的此功能具有多个优点。它允许您从外部在现有对象上创建新过程，并允许将过程调用链接在一起。

Nim支持的另一个范式是功能编程（FP）范式。 FP并不像OOP那样受欢迎，尽管近年来它的受欢迎程度急剧上升。 FP是一种编程风格，主要避免状态更改和使用可变数据。它使用了某些功能，例如一流的功能，匿名功能和闭包，Nim支持所有这些功能。让我们看一个例子，看看以程序样式进行编程与以功能样式进行编程之间的区别。以下代码清单显示了将人们的全名分为姓和名的代码。清单1.7展示了以功能样式完成的操作，清单1.8展示为过程样式。

import sequtils, sugar, strutils
let list = @["Dominik Picheta", "Andreas Rumpf", "Desmond Hume"]
list.map(
    (x: string) -> (string, string) => (x.split[0], x.split[1])
).echo


import strutils
let list = @["Dominik Picheta", "Andreas Rumpf", "Desmond Hume"]
for name in list:
    echo((name.split[0], name.split[1]))
Nim会提前捕捉错误

在本章中，我一直在比较Python和Nim。尽管Nim确实从Python中获得了很多启发，但两种语言在一个重要方面有所不同：Python是动态类型的，而Nim是静态类型的。作为静态类型语言，Nim提供了某种程度的类型安全性，而动态类型编程语言则没有提供这种安全性。尽管Nim是静态类型的，但它感觉非常动态，因为它支持类型推断和泛型。您将在本书的后面详细了解这些功能。目前，将其视为保留动态类型编程语言所允许的高开发速度的一种方法，同时还可以在编译时提供额外的类型安全性。除了静态类型化外，Nim还实现了完全选择加入的异常跟踪机制。使用异常跟踪，您可以确保过程不会引发任何异常，或者仅会从预定义列表中引发异常。通过确保您处理异常，可以防止意外崩溃。

Nim仍需要改善的地方

这个世界上没有什么是完美的，编程语言也不例外。没有一种编程语言可以最可靠，最快速地解决所有问题。每种编程语言都有其优点和缺点，Nim也不例外。到目前为止，我一直专注于Nim的长处。 Nim还有许多我还没有提到的方面，您会在本书中发现它们。但是只谈论Nim的强项是不公平的。 Nim仍然是一种年轻的编程语言，因此它当然仍可以改进。

Nim仍然年轻不成熟

所有编程语言都经历了一段不成熟的时期。 Nim的一些更新和更高级的功能仍然不稳定。使用它们会导致编译器中的错误行为，例如崩溃，尽管崩溃很少发生。重要的是，Nim的不稳定功能已启用，这意味着您不会意外使用它们。 Nim有一个名为Nimble的程序包管理器。在其他编程语言可能有成千上万个软件包的地方，Nim只有约500个。这意味着您可能需要为某些任务编写库。 Nim社区每天都会创建新的程序包，这种情况当然正在改善。在第5章中，我将向您展示如何创建自己的Nimble软件包。

Nim的用户基础和社区仍然很小

与主流编程语言相比，Nim的用户数量很少。结果是几乎没有Nim职位。很难找到一家在生产中使用Nim的公司，但是当这种情况发生时，对好的Nim程序员的需求会使他们的薪水很高。另一方面，关于Nim的最独特的事情之一就是它的开发是非常开放的。 Nim的创建者Andreas Rumpf和其他许多Nim开发人员（包括我）在GitHub和IRC上公开讨论了Nim的未来开发计划。任何人都可以自由挑战这些计划，并且由于社区还很小，所以这样做很容易。 IRC还是向新人提出有关Nim的问题并结识Nim程序员的好地方。
