独自赏晴雨的第一篇博客

github

基本概念

仓库(Repository)

仓库用来存放项目代码，每个项目对应一个仓库，多个开源项目则有多个仓库。

收藏(Star)

收藏项目，方便下次查看

复制克隆项目(Fork)

注意：该fork时的项目是独立存在的。

发起请求 (Pull Request)

发起请求，这个其实是基于Fork的，还是上面那个例子，如果别人在你的基础上做了改进，后来觉得改进的不错，应该把这些改进让更多的人收益，于是就想把自己的改进合并到原有的项目中，这时候它就可以发起一个Pull Request(简称PR),原有项目创建人，也就是你，就可以收到这个请求，这时候你仔细review他的代码，并且测试觉得OK了，就会接受他的PR，这时候他做的改进项目就拥有了。

关注(Watch)

关注项目，当项目更新可以接收到通知。

事务卡片(Issue)

发现代码Bug,但是目前没有成型代码，需要讨论时用

Github主页

账号创建成功或点击网址导航栏github图标都可以进入github主页，该页左侧主要显示用户动态以及关注用户或关注仓库的动态，右侧显示所有的git库。

仓库主页

仓库主页主要显示项目的信息，如，项目代码，版本，收藏/关注/fork情况等。

个人主页

个人信息，头像，个人简介，关注我的人，我关注的人，我关注的git库，我的开源项目，我贡献的开源项目等信息。

Git

Git工作区域

Git Repository (Git 仓库)

最终确定的文件保存到仓库，成为一个新的版本，并且对其他人可见。

暂存区

暂存已经修改的文件最后统一提交到git仓库中。

工作区 （Working Directory)

添加、编辑、修改文件等动作

添加文件流程

Git基础设置

设置用户名

git config --global user.name 'EchoHFC'

设置用户名邮箱

git config --global user.email '3495343534@qq.com'

查看设置

git config --list

向仓库中添加文件

初始化仓库

git init

创建文件

touch a1.py

添加到暂存区

git add a1.py

添加到仓库

git commit -m 'first'

修改仓库文件

  1. vim a1.py
  2. git add a1.py
  3. git commit -m 'xiugai'

删除仓库文件

  1. 删除文件 rm -rf a1.py
  2. 从 Git 中删除文件 git rm a1.py
  3. 提交操作 git commit -m 'shanchu'

Git 管理远程仓库

Git 远程仓库

目的

备份，实现代码共享集中化管理

步骤

克隆项目

git clone 仓库地址

修改文件（同上）

提交到远程仓库

git push

Github Pages 搭建网站

个人站点

访问

https://用户名.github.io

搭建步骤

  1. 创建个人站点 -> 新建仓库（注：仓库名必须是[用户名.github.io]）
  2. 在仓库下新建 index.html的文件即可

Project Pages 项目站点

https://用户名.github.io/仓库名

搭建步骤

  1. 进入项目主页
  2. 在settings页面，点击Launch automatic page generator来自动生成主题界面。
